Introduction to Comparisons

In JS, we have the following mathmatical comparison operators:

  • Greater/less than: a > b, a < b
  • Greater/less than or equal to: a >= b, a <= b
  • Equals: a == b (note that = alone is assign, not equals)
  • Note equals: a != b

In JS, a comparison returns a boolean result. Either 'true' or 'false'. For example:

alert(2 > 1); // Returns true
alert(2 == 1); // Returns false

A comparison result can be assigned to a variable, like any value. For example:

let result = 5 > 4;
alert(result); // Returns true

String Comparisons

JS uses lexicographical order when comparing string to see which is greater than another. Each string is compared letter-by-letter. For example:

Lowercase characters are not equal to uppercase characters. Lowercase are greater; this is based on the characgs Unicode covered later. For example:

Comparison of Different Types

When comparing values of different types, JS conevrs values to numbers. For example:

alert('2' > 1); // true, as string '2' is converted to number 2.
alert('01' == 1); // true, as string '01' is converted to number 1.

For boolean values, 'true' is converted to number 1 and 'false' is converted to number 0. For example:

alert(true == 1); // true
alert(false == 0); // true

Strict Equality

To check if values are equal, use ==.

Note that strict equality has a problem in that it ccannot differentiate 0 from false. For example:

alert(0 == false); // true
// But the same thing happens with an empty string:
alert('' == false); // true

This happens beacause different types are converted to numbers, and just like 'false' boolean, an empty string becomes a zero. If we want to differenate 0 from 'false', use the strict equality operator ===. This checks equality, but unlike ==, === does not convert the type. Therefore:

alert(0 === false); // false

Similarly, non-equality !== does the same for !=.

Comparison of null and undefined

For a strict equality check ===. these a comparison between null and undefined returns false as they're different types. For a non-strict equality check ==, a comparison between the two returns true.

Although both are different types, they are special in that they are not equal to any other value or type, other than each other.

For mathmatical comparisons, null is converted to 0 and undefined is convered to Nan. For example:

alert(null > 0); // false, as null is converted to 0 for mathmatical comparison.
alert(null == 0); // false, as null is NOT converted to 0 for equality comparison.
alert(null >= 0); // true, as null is converted to 0 for mathmatical comparison.

The value undefined should not be compared to other values as it will always be false. For example:

alert(undefined > 0); // false, as undefined is converted to NaN for mathmatical comparison.
alert(undefined < 0); // false, as undefined is converted to NaN for mathmatical comparison.
alert(undefined == 0); // false, as undefined is NOT converted for equality comparison.
