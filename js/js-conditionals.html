Introduction To Conditionals

The following conditional statements are available in js:

  • if
  • if...else
  • if...else if
  • switch

This section covers the first three above. See here for switch statements.

Introduction To The 'if' Statement

Syntax example:

if (condition) {
___// block of code to be executed if the condition is true.
}

Introduction To The 'if else' Statement

Syntax example:

if (condition) {
___// block of code to be executed if the condition is true.
} else {
___// block of code to be executed if the condition is false
}

In summary, the following is happening here:

  • The 'if' keyword is declared, followed by some parethesis.
  • A condition to test is then placed within the parenthesis. Thiw will use comparison operators and will return either true or false.
  • A set of curly braces is next with code which is exectured only if the result of the condition is true.
  • The 'else' keyword is then shown.
  • Finally, another set of curly breaces with code which is executed only if the contidion is false.

Note the if else statements can be writen in a shorter way, but this is not recommended as it makes code less readable. This includes excluding 'else' or curly brackets. For example, the following are perfectly legal, but harder to read:

if (condition) code to run if condition is true
else code to run if condition is false

if (condition) {
___code to run if condition is true
}

Here is an example of a simple if else statement in action.

Introduction To The 'else if' Statement

if (condition1) {
___// block of code to be executed if condition1 is true.
} else if (condition =2) {
___// block of code to be executed if condition1 is false and condition2 is true.
} else {
___// block of code to be executed if both conditions are false. }

Testing Boolean - A Point To Remember

A thing to remember when testing boolean (true/false) values, which comes up a lot, is that any value which is not 'false', or is not 'undefined', 'null', '0', 'NaN' or '', actually returns true. So just a standard string alone, is 'true'. For example:

var cheese = 'Cheddar';

if (cheese) {
___console.log('Yes, it is cheese!');
} else {
___console.log('Nope, not cheese');
}

... so 'Cheddar' returns 'Cheddar' which is true.

Conditionals & Logical Operators

You can use logical operators to test multiple conditions without needing to nest if else statements. When used in conditions:

  • && (AND) - Allows two or more expressions to be chained together so that all must be true for the entire expression to return true.
  • || (OR) - Allows two or more expressions to be chained together so so that one or more have to be true in order for the entire expression to return true.

For example, following on from the nesting example covered earlier, using AND && can prevent the need for nesting:

if (choice === 'sunny' && temparature < 25) {
___para.textContent = 'It is ' + temperature + 'outside. Nice and sunny!';
} else if {

As an example using OR ||, where at lease one expression must be true:

if (iceCreamVanOutside || houseStatus = 'on fire') {
___console.log('Leave the house quickly!');
} else {
___console.log('No need to go outside...');

The NOT logical operator can also be used to negate and expression. Combined with the OR expression above:

if (!(iceCreamVanOutside || houseStatus = 'on fire')) {
___console.log('No need to go outside...');
} else {
___console.log('Leave the house quickly!');
}

In the statement above, OR returns 'true', however NOT negates this and changes the overall express result to 'false'.

When using logical operators, remember to write each expression independantly. For example:

if (x === 5 || 10 || 15) // Will always be true, as '10' and '15' are always true.

if (x === 5 || if x = 10 || if = 15) // Will only be true if x equals 5, 10 or 15.
