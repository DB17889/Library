Data Types

Introduction to Data Types

In JavaScript, there are 7 basic data types:

  1. Number
  2. String
  3. Boolean
  4. "null"
  5. "undefined"
  6. Objects
  7. Symbols

The 'typeof' operator can be used to identify which type of data is stored in a variable. For example:

typeof(x); // or
typeof x;

//returns the data value, e.g."string"
//note that data type 'null' returns 'object' which is an error in the langauge.

This section summarises each data type.

Data Type: Number

A number represents both integer and floating point numbers. There are many operators which can act on numbers, such as *, /, + and -.

As well as regular numbers, there are 'special' number types:

  • Infinity: Represents mathematical infinity. We get this when the number is too large, or as a result of dividing by zero.
  • NaN: Not a Number. Occurs as a result of computational error. For example, dividing a number by a word.

Data Type: String

A string is JS is text surrounded by quotes. There are three types of quotes:

  1. Double quotes: "Hello"
  2. Single quotes: 'Hello'
  3. Backticks: `Hello`

In JS, double quotes and single quotes do the same thing and are both considered as 'simple quotes'. Backticks are 'extended functionality quotes', which allows variables and expressions to be embedded within the string when wrap within ${}. For example:

let name = "Dan";

// embed a variable
alert(`Hello ${name}!`); // Returns: Hello Dan!

// embed an expression
alert(`The result is ${1+1}.`); Returns: The result is 2.

...remember, embedding variables or expressions can only be done when using backticks with ${}.

Data Type: Boolean

A boolean type only has two values: true and false. Most commonly used in yes/no scenerios, where yes = true and no = false.

Data Type: null

The null data type is special type which represents 'unknown' or 'empty'. It's important to remember that null does not mean 'referencing something which does not exist'. For example:

let age = null;

...not that age does exist, it is just empty and unknown.

Data Type: undefined

Like null, undefinred is a special type of its own, where undefined represents 'not yet assigned'. For example, a variable is declared, but not value is assigned to it, such as:

let x;

alert(x); // returns "undefined"

Data Type: Object

The object type is different to those covered so far. Those covered so far are 'primitive' data types as they can only contain a single thing, i.e. a number, string etc. However, objects store a collection of data and compex entities.
