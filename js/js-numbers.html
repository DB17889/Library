Introduction to Numbers

Unlike other programming languages, JS has only one type of number.

A number can be writen with decimals or without decimals. A number that is extra small or extra large can be writen in scientific notation too.

var x = 3.14; // With decimals
var y = 3; // Without decimals
var a = 123e5; // 12300000
var b = 123e-5; // 0.00123

JS doesnt have a variety of number types such integer, floating point etc.; instead JS numbers are always stored as double precision floating point numbers, following international standard IEEE 754.

This standard stores numbers in 64 bits, where the number is stored in bits 0 to 51, the exponent in bits 52 to 62, and the sign in bit 63.

Integers without a decimal or exponent notation are accurate up to 15 digits. For example:

var x = 999999999999999; // x will be 999999999999999
var y = 9999999999999999; // y will be 1000000000000000

The maximum number of decimals in a floating point is 17, but note that numbers and floating point maths isn't always accurate in JS. For example:

var x = 0.2 + 0.1; // x will be 0.30000000000000004

There is a way to overcome this by multiplying then dividing by x, so:

var x = (0.2 * 10 + 0.1 * 10) / 10; // x will be 0.3

Adding Numbers and Strings

Before moving on to addition, remember than in JS, the + operator is used for both addition of numbers and concatenation of string. For example:

var x = 10;
var y = 20;
var z = x = y; // z will equal 30 (a number)

var x = "10";
var y = "20";
var z = x + y; // z will be equal to 1020 (a string)

Bear in mind that adding a number and a string also result in concatenation, for example:

var x = "10"; // a string
var y = 20; // a number
var z = x + y; // z will be 1020 (a string)

But remember, JS works from left to right, so in the example below we have two actions taking place:

var x = 10; // a number
var y = 20; // a number
var z = "30"; // a string
var result = x + y + z; // this will be 3030

...this is because JS, working left to right, first adds the numbers 10 and 20, then concetenates the resulting number of 30 to the sring 30.

Numeric Strings

JS strings can have numeric content, but remember, the + operator concatenates!

For example, this will work:

var x = "100";
var y = "10";
var z = x * y; // z will be 1000.

...as it converts the string to number using multiplication. Also, this will work:

var x = "100";
var y = "10";
var z = x / y; // z will be 10.

...as it converts the string to number using division. This will work also:

var x = "100";
var y = "10";
var z = x - y; // z will be 90.

...as it converts the string to number using subtraction. But this will not work:

var x = "100";
var y = "10";
var z = x + y; // z will be 10010.

...as the + operator will default to concatenate string, not convert string to number.

NaN

In JS, "NaN" is a reserved term which shows that a number is not a real number. Trying to do any calculations with a non-numeric string will result in "NaN".

For example:

var x = 100 / "Apple"; // x will be NaN

Although "NaN" is 'not a number', it's type is: number. For example:

typeof NaN; // returns "number"

Infinity

"Infinity", or "-Infinity", is what JS returns if the number calculated is outside the largest possible number.

"Infinity" is also what is returned when trying to divide by zero. For example:

var x = 2 / 0; // x will be Infinity

Like NaN, Infinity is also type: number. For example:

typeof Infinity; // returns "number"

Hexadecimals

If preceeded by 0x, JS interprets a Hexadecimal value as a number. For example:

var x = 0xFF // x will be 255
